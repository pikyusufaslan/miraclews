based on measured benchmarks · estimate

Server count is max(RAM-bound, CPU-bound, 1) per stack. Idle per-connection RAM is verified on a real NIC — a 2-machine rig, MiracleWS vs tokio-tungstenite vs fastwebsockets under the same workload (benchmark & raw data). MiracleWS measured 0.21 KB/conn (listed 0.25, rounded up; competitors rounded down). Userspace advantage is 40×+; adding the stack-independent kernel/TCP floor (2 KB/conn, still an estimate) brings total-host memory to ~4–6×. The 1 M figures are extrapolated from a slope verified to 200 k connections — not a measured plateau. Throughput is still a loopback benchmark (real-NIC throughput & latency are pending a gigabit rig) — trust the cross-stack ratio, not absolute capacity. Payload bytes are shown for reference but not yet factored into RAM. Validate with your own profiling.
